Build the local-only review UI that satisfies AUDIO-08 (100% review gate) and AUDIO-09 (review UI plays each clip with its label, lets reviewer sign off entry-by-entry):

1. **`tools/tts/review_server.py`** — stdlib `http.server`-based Python server. No Flask/FastAPI/Bottle dependency. Binds 127.0.0.1:8765 (D-19). Routes:
   - `GET /` — server-render HTML page listing every utterance from manifest.yaml with current review status from reviewed.yaml.
   - `GET /audio/{key}` — serves the AAC file at the path from manifest.yaml (after sanity checks: path under `assets/audio/`, no `..`, exists on disk; 404 if missing or not yet generated by Plan 07).
   - `GET /static/{file}` — serves review.css and review.js.
   - `POST /approve/{key}` — atomically updates reviewed.yaml: `reviewed: true`, reviewer (default "Jon" — configurable via env or per-request body field), ISO timestamp, voice, text_hash, notes (optional). Validates with schema.py before write.
   - `POST /rerecord/{key}` — atomically updates pronunciation_overrides.yaml (adds an empty `text:` entry with a `# REVIEW: re-record requested` note, OR a placeholder SSML if Tiro supports it per Plan 01) AND sets `reviewed: false` in reviewed.yaml (with `issue: <text from request>`).
   - `GET /status` — JSON: total utterances, reviewed count, blocked count, last reviewed_at.
   - `POST /shutdown` — gracefully stops the server (used by tests; behind a `--allow-shutdown` flag in production).

2. **`tools/tts/templates/review.html.j2`** — server-side Jinja2 template. Per-row layout:

   Status badge color-coded: ✓ green = approved + hash current, ⚠ yellow = approved but hash drifted, ✗ red = unreviewed, ⏳ gray = re-record queued.

Atomic YAML write helper (used by /approve and /rerecord handlers):
```python
def atomic_write_yaml(path: Path, data: dict) -> None:
    tmp = path.with_suffix(path.suffix + ".tmp")
    tmp.write_text(yaml.safe_dump(data, sort_keys=True, allow_unicode=True))
    os.fsync(tmp.open("rb").fileno())  # or open + fsync via context manager
    tmp.replace(path)  # atomic on POSIX
```
A `threading.Lock` around reads + writes protects against concurrent requests.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → review_server (loopback) | HTTP over 127.0.0.1. No auth (D-19). The reviewer is the user; the server is launched by the user; the trust model is "single-user local tool". |
| review_server → reviewed.yaml + pronunciation_overrides.yaml | Atomic writes; schema validation on every write. |
| review_server → assets/audio/*.aac | Read-only; path-traversal guard required. |
